Why there is a Debug Directory?
In C++ projects, especially when using build systems like Visual Studio, CMake, or Makefiles, a folder named Debug/ is created to hold the compiled output of your project in debug mode.
Build Type	Purpose	Folder
Debug	For development: includes extra info to help with debugging	Debug/
Release	For final distribution: optimized for speed and size	Release/

üîç What‚Äôs Inside the Debug/ Directory?
Typically:

PDB files (.pdb = Program Database) ‚Üí contains symbols and debug info
Other intermediate build files

üõ† What‚Äôs Different About a ‚ÄúDebug Build‚Äù?
Debug builds:
Include symbol information for variables and functions
Disable optimizations to make the code easier to step through
Allow stepping through code line-by-line in a debugger
Are larger and slower than release builds
